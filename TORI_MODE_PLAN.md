# ATLAS Tori Mode Plan

## Purpose

Tori Mode is the relational and environmental interpretation lens inside ATLAS. It exists to help understand whether a school/city ecosystem could support a shared life for Harper and Tori, not merely whether the program is strong for Harper's documentary trajectory.

This document defines the long-term purpose, hierarchy, tone, and interaction model for Tori Mode before further implementation.

Tori Mode should remain part of one coherent atlas. It should not become a duplicate application, a secondary dashboard, or a compressed Program view.

## 1. Core Philosophy

Tori Mode models livability and inhabitation.

It exists to interpret:

- livability
- daily rhythm
- routine friction
- environmental coherence
- relationship sustainability
- healthcare and recoverability
- social/community integration
- creative-life survivability
- long-term sustainable life

It does not exist to optimize admissions, rank programs, or model documentary-professional trajectory.

The core frame is not: "Which MFA is best?"

The core frame is: "Could a real life be sustained here, for two creative people, under ordinary pressure?"

## 2. Primary Questions

Tori Mode should consistently answer or preserve uncertainty around questions such as:

- Could we live here?
- Would Tori have a life here?
- Would this environment support the relationship?
- Would daily life feel sustainable?
- Would social and community infrastructure exist?
- Would healthcare and recoverability be viable?
- Would this environment support creative-life continuity?
- Would routine friction quietly exhaust the relationship?
- Would the place offer enough independence for Tori, not only enough opportunity for Harper?
- Would this ecosystem support a stable future after the first semester glow fades?

The system should be comfortable saying: "not enough evidence yet."

Unresolved life-fit questions are not failures of the model. They are part of the model.

## 3. Relationship To Harper Mode

Harper Mode focuses on documentary MFA fit:

- program fit
- documentary ecosystem
- mentorship
- curriculum
- artistic and professional trajectory
- institutional metabolism
- authorship formation
- labor ecology

Tori Mode focuses on lived-environment fit:

- life fit
- inhabitation
- environmental sustainability
- relationship durability
- daily rhythm
- social/community viability
- healthcare and recoverability
- creative-life survivability

Tori Mode is not:

- a duplicate app
- a second Program dashboard
- a stripped-down Harper Mode
- a metrics panel
- a generic partner-accessibility mode
- a productivity dashboard

Both modes should feel like the same ATLAS world interpreted through different priorities.

Harper asks: "Can this institution form the filmmaker?"

Tori asks: "Can this place hold the life around the filmmaker?"

## 4. Information Hierarchy

### Tori Mode Should Foreground

- Livability Score as an interpretive life-fit signal
- concise environmental synthesis
- strongest Environmental Anchors
- community infrastructure
- queer/community infrastructure where relevant
- daily rhythm and routine feasibility
- relationship/life-fit interpretation
- healthcare/recoverability
- social ecology
- creative-life survivability
- unresolved environmental risk

### Tori Mode Should De-Emphasize

- admissions competitiveness
- detailed program scoring
- documentary-professional infrastructure
- institutional prestige
- dense submetrics
- curriculum detail
- program-page summaries
- exposed model machinery

Program information may still exist in ATLAS, but it should not dominate Tori Mode surfaces.

## 5. Interaction Philosophy

Tori overview cards should be informational environmental surfaces, not navigation hubs.

They should prioritize:

- image-first composition
- school/city identity
- one primary life-fit signal
- one concise synthesis line
- selected anchors
- a clear Environmental Context action

They should not behave like Program cards by default.

### Overview vs Modal

Overview cards answer: "What is the life-fit read at a glance?"

Environmental Context modals answer: "What lived-environment evidence and interpretation produce that read?"

### Progressive Disclosure

Tori Mode should hide detail until it is useful.

The overview should not expose:

- metric grids
- stacked subcategory panels
- long synthesis paragraphs
- multiple competing radials
- raw category lists

Detailed context belongs in the modal or a future Environment page.

### Modal Interaction

The Environmental Context modal should:

- open without route changes
- render at page level
- close reliably
- keep the overview grid stable
- replace content when another card's context is opened
- preserve the cinematic overview underneath

## 6. Visual Direction

Tori Mode should preserve ATLAS's cinematic and editorial language.

Preserve:

- environmental mood
- city photography
- restrained overlays
- negative space
- documentary archive feeling
- calm visual rhythm
- interpretive atmosphere
- soft hierarchy

Avoid:

- dashboard aesthetics
- metrics overload
- productivity-app UI
- stacked analytics panels
- generic accessibility/settings-mode styling
- "Harper-lite" design
- travel-guide cards
- system-debug language

Tori Mode should feel as intentional and emotionally weighted as Harper Mode, not secondary to it.

## 7. Environmental Context Modal

The Environmental Context modal is the main contextual depth mechanism for Tori Mode.

Its purpose is to provide a compact environmental reading without turning the overview card into a detail page.

It should include:

- full environmental synthesis
- life-fit interpretation
- selected anchors
- community/social integration cues
- relationship sustainability cues
- healthcare/recoverability cues
- daily friction notes
- unresolved uncertainty where appropriate

It should feel like:

- interpretive field notes
- lived-context synthesis
- environmental reading
- a documentary atlas annotation

It should not feel like:

- a second detail page
- a metrics dashboard
- a data dump
- a program summary
- a modal spreadsheet

Future expansions may include:

- epistemic status cues
- richer anchor metadata
- compact contradiction blocks
- selected image/media support
- links into the future Environment page

## 8. Future System Integration

### Environment / City & Life

Tori Mode should draw from the shared Environment / City & Life system, not maintain a separate data model.

Profile distinction should come from:

- weighting
- placement
- visibility
- language
- emphasis

—not separate architectures.

### Ontology Integration

Future ontology work can clarify relationships among inhabitation, environmental anchors, labor ecology, contradiction, and institutional metabolism. It should not be merged into live rendering until the user-facing language is stable.

### Epistemic Rendering

Tori Mode is especially dependent on inferred and synthesized claims. It should use subtle epistemic cues to distinguish:

- verified evidence
- inferred environmental reads
- synthesis
- institutional claims
- speculative interpretation
- unresolved ambiguity

The goal is felt epistemic texture, not academic markup.

### Relationship Modeling

Future relationship modeling should remain human-centered. It should not reduce the relationship to compatibility scores. It should interpret pressures, supports, independence, recovery, shared routine, and long-term viability.

### Environmental Scoring

Livability Score should remain transparent, low-precision, and interpretive. It should support synthesis rather than replace it.

### Topology Systems

Future topology systems may map relationships among programs, cities, neighborhoods, anchors, documentary-world nodes, and social infrastructure. Tori Mode should benefit from topology without becoming a graph interface.

### Narrative Synthesis

Future narrative synthesis should produce concise, human-readable environmental reads. It should avoid placeholder shorthand and score-speak.

### Media / Imagery

Tori Mode should use imagery to carry mood, place, and environmental texture. Media should not become a tourism gallery.

## 9. Tone And Style Guidance

Tori synthesis language should feel:

- observational
- human-centered
- environmentally aware
- emotionally intelligent
- restrained
- documentary-oriented
- specific without overclaiming

Avoid:

- score-speak
- optimization jargon
- placeholder shorthand
- generic strong/weak phrasing
- institutional marketing tone
- relocation-blog tone
- automated-sounding summaries

Good direction:

- "Compact daily rhythm with enough cultural anchors to support a shared routine, though healthcare and state-level stress remain open questions."
- "Strong city texture and social possibility, but routine fragmentation and cost pressure would need careful management."
- "Recoverable small-city pace with visible third places and modest arts infrastructure, best read as a life-building environment rather than a spectacle city."

Bad direction:

- "California strong."
- "Major concern."
- "Good support."
- "Best livability option."
- "Optimized for relationship sustainability."

## 10. Architectural Guidance

Future implementation should keep Tori Mode lightweight.

Priorities:

- preserve shared atlas architecture
- reuse the Environment / City & Life adapter
- preserve modal-driven contextual depth
- avoid duplicate rendering systems
- avoid separate Harper/Tori data models
- avoid turning profile differences into separate applications
- keep overview cards cinematic and stable
- keep detail in modals or future Environment pages
- preserve static GitHub Pages compatibility

Tori Mode should evolve as a lens, not a fork.

## Implementation Principle

Before adding new UI, ask:

1. Does this help answer whether a life can be sustained here?
2. Does it belong in Tori overview, Environmental Context, Program detail, or the future Environment page?
3. Does it preserve the shared atlas architecture?
4. Does it reduce or increase dashboard clutter?
5. Does it make Tori Mode more emotionally and environmentally legible?

If the answer is unclear, preserve restraint and defer implementation.
